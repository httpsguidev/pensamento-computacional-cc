## Contexto e Objetivos

Este repositório contém a entrega da atividade prática "A Arquitetura da Resiliência". O objetivo é aplicar os conceitos de identificação, correção e avaliação de erros (debugging estruturado) em um projeto real. 

A aplicação base utilizada é o **MCG Auto**, um sistema backend (Java/Spring Boot) focado na gestão de concessionárias e oficinas automotivas. Para este exercício, isolamos a regra de negócio central do módulo de vendas: o **Processamento de Venda de Veículo**.

O laboratório demonstra a transição de um código frágil (com falhas de ambiguidade, erros lógico-matemáticos e sem ação defensiva) para um código resiliente e seguro.
