# pensamento-computacional-cc

Este repositório documenta a aplicação dos pilares do pensamento computacional para a estruturação de um sistema de aprendizado eficiente. O objetivo é transformar o desafio de equilibrar a transição de carreira, o estágio em desenvolvimento e a vida acadêmica em uma solução lógica, escalável e automatizável.

# O Problema: "Otimização do Roadmap de Desenvolvimento Back-end"

Como estudante e estagiário de software, a coexistência de diferentes stacks (Java/Spring no roadmap pessoal e Python/SQL no estágio) gera uma alta carga cognitiva. O problema reside em como reformular essa sobrecarga em um fluxo de execução que garanta a progressão constante sem comprometer a eficiência ou atingir o esgotamento (burnout).

---

## 1. Decomposição

*Conceito: Processo de dividir um único problema em partes menores com a intenção de facilitar a maneira como ele será resolvido e gerenciado.*

Para estruturar este roadmap híbrido, o desafio foi decomposto nos seguintes módulos:

- **Mapeamento de Dependências (Java/Spring):** Divisão do roadmap principal em camadas de complexidade, como Fundamentos, Spring Boot Essentials, Segurança/OAuth2 e Arquitetura.  
- **Integração de Contextos (Estágio):** Acoplamento dos estudos de Python e SQL diretamente nas janelas de execução do estágio, utilizando o aprendizado prático como validação de conhecimento.  
- **Gestão de Janelas de Tempo:** Segmentação do dia em blocos de processamento distintos, facilitando o agendamento na presença de incertezas.  
- **Validação de Saída (Output):** Implementação de instâncias menores do problema (mini-projetos e exercícios de lógica) para validar cada módulo de forma isolada.

---

## 2. Abstração

*Conceito: Consiste na filtragem da classificação dos dados, separando apenas os elementos essenciais e ignorando detalhes irrelevantes para que o problema seja mais fácil de entender.*

Neste planejamento, foram aplicados filtros para focar na essência do progresso técnico:

- **Abstração de Ferramental:** Focar na compreensão da lógica do algoritmo e arquitetura, em vez de memorizar sintaxes mecânicas ou configurações estéticas de IDEs.  
- **Foco no Conceito Transversal:** Identificar padrões que se repetem entre linguagens, como a conceptualização e ideias fundamentais, abstraindo diferenças superficiais de sintaxe.  
- **Redução de Ruído:** Escolher representações apropriadas para o roadmap, ignorando temporariamente tecnologias que não compõem o Core atual para evitar o desperdício de processamento mental.

---

## Reflexão e Estratégia

O pensamento computacional é uma habilidade analítica fundamental que nos permite usar a inteligência humana para formular soluções que podem ser automatizadas com eficiência. Ao tratar o aprendizado como um sistema de processamento de informação, aplico estratégias de prevenção e proteção contra sobrecarga, garantindo que o sistema de estudo seja resiliente e de longo prazo.

---

## Conexão com o Desenvolvimento

Esta forma de pensar é a mesma utilizada no cotidiano de um desenvolvedor:

- **Modelagem de Dados:** Escolher representações apropriadas para tornar problemas complexos tratáveis.  
- **Arquitetura de Sistemas:** Modularizar o conhecimento e as tarefas em antecipação a novos desafios técnicos, criando um "cache" de competências prontas para uso futuro.
